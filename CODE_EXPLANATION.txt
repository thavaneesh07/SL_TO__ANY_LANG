PROJECT OVERVIEW
================================================================================
Sign2Kannada is a real-time hand gesture recognition system that:
- Detects hand gestures (digits 0-9) from webcam video in real-time
- Extracts hand landmarks using MediaPipe's hand detection
- Classifies gestures using a trained machine learning model (Random Forest)
- Translates recognized digits into Kannada language text
- Displays live predictions with confidence scores on screen

Current Status: FULLY FUNCTIONAL
- All core modules complete and tested
- Real-time inference working on webcam
- Model training pipeline operational


TECHNOLOGY STACK
================================================================================
Python 3.14.6

Core Libraries:
- MediaPipe: Hand detection and landmark extraction
- OpenCV (cv2): Video capture and frame processing
- scikit-learn: Machine learning model (Random Forest Classifier)
- NumPy: Numerical computations
- PIL/Pillow: Kannada font rendering with text effects
- joblib: Model serialization and loading


ARCHITECTURE & PIPELINE
================================================================================

1. DATA PREPARATION PHASE (preprocess.py)
   └─> Reads images from data/dataset/<digit>/ folders
   └─> Extracts 63-dimensional hand landmark vectors per image
   └─> Writes processed data to data/landmarks.csv

2. MODEL TRAINING PHASE (train.py)
   └─> Loads landmark data from CSV
   └─> Splits into 80% training / 20% testing
   └─> Trains Random Forest Classifier (200 trees)
   └─> Evaluates with cross-validation and test metrics
   └─> Saves trained model to model.pkl

3. REAL-TIME INFERENCE PHASE (main.py)
   └─> Loads pre-trained model.pkl
   └─> Opens webcam stream
   └─> For each frame:
       - Extracts hand landmarks
       - Predicts digit class
       - Applies stability filters (majority voting)
       - Translates to Kannada
       - Renders on-screen overlay
   └─> Displays live predictions with confidence bars


FILE-BY-FILE BREAKDOWN
================================================================================

1. extractor.py
   ────────────────────────────────────────────────────────────────────────
   PURPOSE: Hand landmark extraction engine
   
   KEY CLASS: MediaPipeExtractor
   - Dual backend support:
     * MediaPipe Solutions (if available) - classical approach
     * MediaPipe Tasks (fallback) - modern task-based approach
   - NO hand preference: Both LEFT and RIGHT hands are treated equally
   - All detected hands are processed independently
   
   KEY METHODS:
   - __init__(mode=True/False): Initialize with static or live mode
   - extract(frame): Extract landmarks from first detected hand
     * DEPRECATED: Use extract_all() for production
     * Kept for backward compatibility with preprocess.py
   - extract_all(frame): Extract all detected hands (primary method)
     * Returns list of dicts: [{"label": "Left"|"Right", "landmarks": vector}]
     * Both hands processed with equal priority
   - draw_all_hands(frame): Draw both left and right hand skeletons
   - draw_hand_by_index(frame, idx): Draw specific hand by index
   - draw_first_hand(frame): DEPRECATED - use draw_hand_by_index(frame, 0)
   - _normalize_landmarks(): Make features position/scale-invariant
   
   OUTPUT FORMAT: 63-dimensional vector per hand
   - 21 hand landmarks × 3 coordinates (x, y, z)
   - Normalized by wrist position and hand scale
   - Makes gestures scale/position-invariant
   
   HAND PROCESSING STRATEGY (EQUAL TREATMENT):
   ✓ No hardcoded preference for right hand
   ✓ Both hands extracted and processed independently
   ✓ Each hand assigned "Left" or "Right" label from MediaPipe
   ✓ Returns all detected hands in order (no filtering/selection)
   ✓ Caller (main.py) decides how to use multiple hands


2. preprocess.py
   ────────────────────────────────────────────────────────────────────────
   PURPOSE: Build training dataset from raw images
   
   WORKFLOW:
   1. Scans data/dataset/ folder for numeric label directories (0-9)
   2. Reads all images (.jpg, .png, .bmp, .webp) from each label folder
   3. Extracts 63-dimensional landmarks using MediaPipeExtractor(mode=True)
   4. Filters out zero-vectors (no hand detected)
   5. Writes valid samples to data/landmarks.csv
   
   OUTPUT FILE: data/landmarks.csv
   Format: CSV with header "label,x0,y0,z0,...,x20,y20,z20"
   - 64 columns total (1 label + 63 coordinates)
   - One row per valid training sample
   
   STATISTICS PRINTED:
   - Number of processed samples
   - Number of skipped samples (no hand detected)
   - Total images scanned


3. train.py
   ────────────────────────────────────────────────────────────────────────
   PURPOSE: Train classifier model on extracted landmarks
   
   WORKFLOW:
   1. Load data/landmarks.csv
   2. Validate data quality:
      - Correct shape (64 columns)
      - No NaN values
      - At least 2 distinct classes
      - Minimum 2 samples per class
   3. Split: 80% training / 20% testing (stratified)
   4. Train RandomForestClassifier:
      - 200 decision trees
      - max_features="sqrt" (standard for classification)
      - class_weight="balanced" (handles imbalanced data)
      - Uses all CPU cores (n_jobs=-1)
   5. Evaluate:
      - 5-fold cross-validation on training data
      - Classification report on test set
      - Compute test accuracy
   6. Save model to model.pkl
   
   CLASSIFIER CHOICE: Random Forest
   - Why not KNN? 
     * KNN is O(n) inference (slow with many samples)
     * Distance metrics sensitive to feature scaling
   - Why Random Forest?
     * Fast inference: O(tree depth)
     * Robust to hand orientation variance
     * Returns well-calibrated probability estimates
     * Good generalization from relatively small training sets
     * Inherently handles multi-class problems


4. translator.py
   ────────────────────────────────────────────────────────────────────────
   PURPOSE: Mapping between digit predictions and Kannada text
   
   DICTIONARY: DIGIT_TO_KANNADA
   Mapping:
   - "0" → "ಸೊನ್ನೆ"  (Sonne - Zero)
   - "1" → "ಒಂದು"    (Ondu - One)
   - "2" → "ಎರಡು"    (Eradu - Two)
   - "3" → "ಮೂರು"    (Mooru - Three)
   - "4" → "ನಾಲ್ಕು"  (Nalku - Four)
   - "5" → "ಐದು"     (Aidu - Five)
   - "6" → "ಆರು"     (Aaru - Six)
   - "7" → "ಏಳು"     (Eelu - Seven)
   - "8" → "ಎಂಟು"    (Entu - Eight)
   - "9" → "ಒಂಬತ್ತು" (Ombattu - Nine)


5. main.py
   ────────────────────────────────────────────────────────────────────────
   PURPOSE: Real-time webcam gesture recognition with live display
   
   WORKFLOW:
   1. Load model.pkl (Random Forest)
   2. Resolve Kannada font path (tries multiple locations)
   3. Open webcam (1280x720 resolution requested)
   4. Main loop per frame:
      a) Extract landmarks from all detected hands
      b) Draw hand skeleton overlay
      c) Get probability predictions from model
      d) Apply stability filters:
         - Majority voting over last 9 frames (HISTORY_LEN)
         - Require minimum 3 votes to display (MIN_VOTES)
         - Confidence threshold 0.28 (CONF_THRESHOLD)
      e) Track hand presence/absence
      f) Format and render text:
         - Digit (English) - white text
         - Kannada word - dark with white outline (PIL)
         - Confidence bar (color-coded: green/amber/red)
         - FPS counter
      g) Handle timeouts (reset display if no hand for 45 frames)
   
   KEY HYPERPARAMETERS:
   - HISTORY_LEN=9: Frames for majority voting
   - MIN_VOTES=3: Min votes to declare digit stable
   - CONF_THRESHOLD=0.28: Min model confidence (low due to domain gap)
   - NO_HAND_TIMEOUT=15: Frames before resetting display
   - COMMIT_COOLDOWN_SEC=0.35: Prevent duplicate detections when holding
   - NUMBER_RESET_NO_HAND_FRAMES=45: Clear number after extended no-hand
   
   RENDERING FUNCTIONS:
   - draw_kannada_text(): PIL-based Kannada rendering with outline
   - draw_confidence_bar(): Color-coded confidence visualization
   - draw_top3(): Debug view of top 3 predictions
   - draw_fps(): Frame rate display
   
   COLOR SCHEME (BGR):
   - WHITE: (255, 255, 255)
   - BLACK: (20, 20, 20)
   - GREEN: (0, 200, 0) - High confidence (≥65%)
   - AMBER: (0, 165, 255) - Medium confidence (45-65%)
   - RED: (0, 0, 220) - Low confidence (<45%)
   - CYAN: (220, 220, 0) - FPS counter
   
   USER CONTROLS:
   - Press 'q' to quit application

KEY FEATURES & STABILITY MECHANISMS
================================================================================

1. HAND LANDMARK NORMALIZATION (extractor.py)
   - Translates by wrist position (position-invariant)
   - Scales by wrist-to-index distance (scale-invariant)
   - Makes same gesture work anywhere on screen, any distance
   
2. CONFIDENCE FILTERING (main.py)
   - Model probability threshold: 0.28
   - Prevents false positives from random detections
   - Low threshold because ML domain gap exists
   
3. MAJORITY VOTING SMOOTHING (main.py)
   - Tracks last 9 frame predictions
   - Requires 3 matching votes to display
   - Filters out temporal jitter and single-frame noise
   
4. HAND TIMEOUT HANDLING (main.py)
   - Resets display if no hand detected for 15 frames
   - Prevents stale predictions lingering
   - Clears number buffer after 45 frames no-hand
   
5. DEBOUNCE PROTECTION (main.py)
   - Cooldown of 0.35 seconds between number commits
   - Prevents holding a gesture from triggering multiple detections
   - Simulates intentional gesture vs accidental holding
   
6. FPS MONITORING
   - Tracks frame rate over last 30 frames
   - Displays on screen for performance debugging
   - Helps identify bottlenecks

DEBUGGING NOTES
================================================================================

If model shows low accuracy:
- Collect more training samples
- Verify images are clear and well-lit
- Check that landmarks are being extracted (preprocess shows non-zero processed count)
- Consider hand position/orientation variance in dataset

If real-time recognition is unstable:
- Increase HISTORY_LEN (e.g., 11-13) for longer smoothing window
- Increase MIN_VOTES requirement
- Adjust CONF_THRESHOLD (higher = stricter)
- Check webcam brightness/lighting

If Kannada text doesn't render:
- Verify font file exists at assets/fonts/NotoSansKannada*.ttf
- Check font file is readable and not corrupted
- Ensure Pillow is installed correctly

If webcam doesn't open:
- Verify webcam device at index 0 (or adjust cv2.VideoCapture(N))
- Check camera permissions in OS settings
- Confirm no other app is monopolizing the camera
